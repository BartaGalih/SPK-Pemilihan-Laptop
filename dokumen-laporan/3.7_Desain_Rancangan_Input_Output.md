# 3.7 Desain Rancangan Input dan Output

Bagian ini menjelaskan rancangan antarmuka **input** (formulir untuk memasukkan/mengubah data) dan **output** (tampilan hasil/laporan) pada aplikasi SPK Rekomendasi Laptop. Aplikasi memakai **kriteria dinamis (EAV)**: kriteria dapat ditambah/dihapus, sehingga form laptop dan tabel hasil **mengikuti kriteria yang aktif** (bukan kolom tetap).

---

## A. DESAIN RANCANGAN INPUT

### A.1 Rancangan Input Data Laptop (Tambah / Edit)

Form input laptop bersifat **dinamis**: selalu ada field Nama, lalu **satu input nilai untuk setiap kriteria aktif**. Jika kriteria bertambah/berkurang, jumlah input ikut menyesuaikan.

```
+--------------------------------------------------------------+
|  Tambah Laptop                                               |
+--------------------------------------------------------------+
|  Nama Laptop *                                              |
|  [____________________________________________________]      |
|                                                              |
|  --- Nilai per Kriteria (mengikuti kriteria aktif) ---       |
|                                                              |
|  Harga (Rp) * [Cost]           RAM (GB) * [Benefit]          |
|  [ _______________ ]           [ _______________ ]           |
|                                                              |
|  CPU Score (pts) * [Benefit]   Bobot (kg) * [Cost]           |
|  [ _______________ ]           [ _______________ ]           |
|                                                              |
|  Storage (GB) * [Benefit]      Baterai (jam) * [Benefit]     |
|  [ _______________ ]           [ _______________ ]           |
|                                                              |
|              ( ...input tambahan jika kriteria ditambah... ) |
|                                                              |
|                          [ Batal ]   [ Simpan ]             |
+--------------------------------------------------------------+
```

**Tabel Rancangan Field Input – Data Laptop**

| No | Nama Field | Label | Tipe Input | Validasi / Ketentuan |
|----|------------|-------|------------|----------------------|
| 1 | name | Nama Laptop | Text | Wajib, maks. 200 karakter |
| 2 | values[criteria_id] | (satu per kriteria aktif) | Number | Wajib, numerik, minimal 0 — satu input untuk tiap kriteria |

> Nilai disimpan ke tabel `laptop_values` (EAV) sebagai pasangan (laptop, kriteria, nilai). Setiap input diberi label nama kriteria + satuan + badge tipe (Benefit/Cost).

---

### A.2 Rancangan Input Kriteria (Tambah / Edit)

Form untuk membuat kriteria baru atau mengubah kriteria. Input bobot **dibatasi maksimal sisa bobot** yang tersedia agar total tidak melebihi 1,00.

```
+--------------------------------------------------------------+
|  Tambah Kriteria Baru                                        |
+--------------------------------------------------------------+
|  (!) Total bobot sudah 1.00 → kurangi bobot kriteria lain    |
|      dulu sebelum menambah (muncul bila sisa = 0)            |
|                                                              |
|  Nama Kriteria *                                            |
|  [ Kualitas Layar__________________________________ ]        |
|                                                              |
|  Satuan                         Tipe *                       |
|  [ nits____________ ]           [ Benefit / Cost  ▼ ]        |
|                                                              |
|  Bobot (NBF) *                                              |
|  [ 0.10____________ ] [ 10% ]                               |
|  Sisa bobot yang tersedia: 0.10  (maksimal yang boleh diisi) |
|  (!) Bobot melebihi sisa → peringatan langsung muncul        |
|                                                              |
|                          [ Batal ]   [ Simpan ]             |
+--------------------------------------------------------------+
```

**Tabel Rancangan Field Input – Kriteria**

| No | Nama Field | Label | Tipe Input | Validasi / Ketentuan |
|----|------------|-------|------------|----------------------|
| 1 | name | Nama Kriteria | Text | Wajib, maks. 100 karakter |
| 2 | unit | Satuan | Text | Opsional, maks. 20 karakter |
| 3 | type | Tipe | Pilihan (benefit/cost) | Wajib |
| 4 | weight | Bobot (NBF) | Number (desimal) | Wajib, 0,01 – **sisa bobot tersedia** (maks. agar total ≤ 1,00) |

> Indikator persentase dan peringatan "melebihi sisa" diperbarui secara langsung (real-time) saat mengetik.

---

### A.3 Rancangan Input Simpan Arsip (Snapshot)

Muncul sebagai dialog (modal) di halaman Hasil Rekomendasi untuk menyimpan salinan hasil.

```
+--------------------------------------------------+
|  Simpan Hasil ke Arsip                       [x] |
+--------------------------------------------------+
|  Yang disimpan adalah SALINAN (laptop, kriteria, |
|  bobot, dan hasil hitung saat ini).              |
|                                                  |
|  Judul Arsip *                                  |
|  [ Rekomendasi Mahasiswa — Juni 2026________ ]   |
|                                                  |
|  Catatan                                        |
|  [ ________________________________________ ]    |
|                                                  |
|                      [ Batal ]   [ Simpan Arsip ]|
+--------------------------------------------------+
```

| No | Nama Field | Label | Tipe Input | Validasi |
|----|------------|-------|------------|----------|
| 1 | title | Judul Arsip | Text | Wajib, maks. 150 karakter |
| 2 | note | Catatan | Textarea | Opsional, maks. 500 karakter |

```
| #  | Nama Laptop    | Harga    | RAM  | CPU Score | Bobot | Storage |Baterai| Aksi |

| 2  | HP OmniBook    |15289000  |  16  |   19643   | 0.9   |   512   |  15   |👁 ✎ 🗑|
```

### B.3 Rancangan Output Kriteria & Bobot

Daftar kriteria + indikator total bobot dan sisa.

```
+--------------------------------------------------------------------------+
|  Manajemen Kriteria & Bobot (NBF)             [ + Tambah Kriteria ]       |
+--------------------------------------------------------------------------+
|  ✓ Total bobot saat ini: 1.00 — siap untuk dihitung                      |
|  (atau)  (!) Total 0.90 — sisa 0.10. Sesuaikan agar total = 1.00         |
+------+--------------+--------+---------+-----------+-------------+--------+
| Kode | Nama Kriteria| Satuan | Tipe    | Bobot(NBF)| Proporsi    | Aksi   |
+------+--------------+--------+---------+-----------+-------------+--------+
| C1   | Harga        | Rp     | Cost ↓  |   0.30    | ███░░ 30%   | ✎  🗑   |
| C2   | RAM          | GB     | Benefit↑|   0.20    | ██░░░ 20%   | ✎  🗑   |
| ...  (kriteria dapat ditambah / dihapus)                                 |
+------+--------------+--------+---------+-----------+-------------+--------+
```

> Tombol Hapus melakukan **soft delete** (kriteria diarsipkan, tidak benar-benar hilang dari basis data) sehingga snapshot/arsip tetap konsisten.

### B.4 Rancangan Output Hasil Rekomendasi (MFEP)

Terdiri dari kartu **Top 3**, **matriks evaluasi (kolom kriteria dinamis)**, dan **tabel bobot**. Tombol **Hitung dinonaktifkan** bila total bobot ≠ 1,00, dan tersedia tombol **Simpan ke Arsip**.

```
+--------------------------------------------------------------------------+
|  Hasil Rekomendasi MFEP            [ Simpan ke Arsip ] [ Hitung MFEP ]    |
|  (!) Tombol hitung nonaktif bila total bobot != 1.00                     |
+--------------------------------------------------------------------------+

  ┌─ #1 🏆 Terbaik ─┐  ┌─ #2 🥈 Runner-up ┐  ┌─ #3 ⭐ Pilihan ke-3 ┐
  │ ASUS Vivobook   │  │ HP OmniBook      │  │ ASUS Vivobook M..  │
  │ TBE     4.5350  │  │ TBE     4.2275   │  │ TBE     4.1150     │
  │ ██████████      │  │ █████████░       │  │ █████████░         │
  └─────────────────┘  └──────────────────┘  └────────────────────┘

  Matriks Evaluasi Lengkap  (kolom NEF/NBE mengikuti kriteria aktif)
+------+--------------+-----+-----+-----+-----+ ... +--------+
| Rank | Nama Laptop  |  C1       |  C2       | ... |  TBE   |
|      |              | NEF | NBE | NEF | NBE | ... |        |
+------+--------------+-----+-----+-----+-----+ ... +--------+
|  1   | ASUS Vivobook|4.55 |0.91 |5.00 |1.00 | ... | 4.5350 |

  Bobot Kriteria (NBF) yang Digunakan   ← tabel bobot saat perhitungan
```

## C. Ringkasan Rancangan Input & Output

| Kode | Jenis | Nama Rancangan | Halaman Terkait |
|------|-------|----------------|-----------------|
| I-1 | Input | Form Data Laptop (dinamis, Tambah/Edit) | /laptops/create, /laptops/{id}/edit |
| I-2 | Input | Form Kriteria (Tambah/Edit, bobot maks = sisa) | /criteria/create, /criteria/{id}/edit |
| I-3 | Input | Dialog Simpan Arsip | /results (modal → POST /results/snapshot) |
| O-1 | Output | Daftar Data Laptop (kolom dinamis) | /laptops |
| O-2 | Output | Detail Laptop | /laptops/{id} |
| O-3 | Output | Daftar Kriteria & Bobot | /criteria |
| O-4 | Output | Hasil Rekomendasi MFEP | /results |
| O-5 | Output | Daftar Arsip Hasil | /snapshots |
| O-6 | Output | Detail Arsip | /snapshots/{id} |
