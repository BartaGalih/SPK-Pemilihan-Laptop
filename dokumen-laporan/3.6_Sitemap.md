# 3.6 Sitemap (Peta Situs)

Sitemap adalah peta struktur halaman aplikasi yang menggambarkan seluruh halaman yang dapat diakses pengguna beserta hubungan/alur navigasinya. Aplikasi **SPK Rekomendasi Laptop** tidak menggunakan sistem login, sehingga seluruh halaman dapat diakses langsung melalui menu sidebar.

## 3.6.1 Struktur Halaman (Hierarki)

```
SPK Rekomendasi Laptop
│
├── Beranda ( / )
│     └── Otomatis dialihkan ke halaman Data Laptop
│
├── [Menu 1] Data Laptop ( /laptops )
│     ├── Tambah Laptop          ( /laptops/create )      → form nilai dinamis per kriteria
│     ├── Detail Laptop          ( /laptops/{id} )
│     ├── Edit Laptop            ( /laptops/{id}/edit )
│     └── (aksi) Hapus Laptop    → kembali ke Data Laptop
│
├── [Menu 2] Kriteria & Bobot ( /criteria )
│     ├── Tambah Kriteria        ( /criteria/create )     → bobot dibatasi sisa yang tersedia
│     ├── Edit Kriteria          ( /criteria/{id}/edit )
│     └── (aksi) Hapus Kriteria  → soft delete, kembali ke Kriteria & Bobot
│
├── [Menu 3] Hasil Rekomendasi ( /results )
│     ├── (aksi) Hitung MFEP     ( /results/calculate )   → nonaktif bila total bobot ≠ 1.00
│     └── (aksi) Simpan ke Arsip ( /results/snapshot )    → menyimpan salinan hasil
│
└── [Menu 4] Arsip Hasil ( /snapshots )
      ├── Detail Arsip           ( /snapshots/{id} )
      └── (aksi) Hapus Arsip     → kembali ke Arsip Hasil
```

## 3.6.2 Daftar Halaman & Keterangan

| No | Halaman | URL | Akses Dari | Fungsi |
|----|---------|-----|------------|--------|
| 1 | Beranda | `/` | URL langsung | Halaman awal, otomatis dialihkan ke Data Laptop |
| 2 | Data Laptop | `/laptops` | Sidebar → Data Laptop | Daftar laptop + nilai tiap kriteria (kolom dinamis) |
| 3 | Tambah Laptop | `/laptops/create` | Tombol "Tambah Laptop" | Form input laptop; satu input nilai per kriteria aktif |
| 4 | Detail Laptop | `/laptops/{id}` | Tombol "Detail" | Spesifikasi laptop + hasil MFEP-nya |
| 5 | Edit Laptop | `/laptops/{id}/edit` | Tombol "Edit" | Form ubah data laptop (dinamis) |
| 6 | Kriteria & Bobot | `/criteria` | Sidebar → Kriteria & Bobot | Daftar kriteria + indikator total bobot |
| 7 | Tambah Kriteria | `/criteria/create` | Tombol "Tambah Kriteria" | Form kriteria baru; bobot maks = sisa tersedia |
| 8 | Edit Kriteria | `/criteria/{id}/edit` | Tombol "Edit" | Form ubah kriteria & bobot |
| 9 | Hasil Rekomendasi | `/results` | Sidebar → Hasil Rekomendasi | Peringkat, Top 3, matriks NEF/NBE (dinamis) |
| 10 | Arsip Hasil | `/snapshots` | Sidebar → Arsip Hasil | Daftar hasil perhitungan yang disimpan |
| 11 | Detail Arsip | `/snapshots/{id}` | Tombol "Lihat" | Salinan kriteria, laptop, nilai, & hasil pada saat disimpan |

## 3.6.3 Aksi (Bukan Halaman)

Beberapa proses berjalan sebagai aksi (tanpa halaman tersendiri) lalu mengembalikan pengguna ke halaman terkait:

| Aksi | Method/URL | Hasil |
|------|------------|-------|
| Simpan laptop baru | `POST /laptops` | Kembali ke Data Laptop |
| Perbarui laptop | `PUT /laptops/{id}` | Kembali ke Data Laptop |
| Hapus laptop | `DELETE /laptops/{id}` | Kembali ke Data Laptop |
| Simpan kriteria baru | `POST /criteria` | Kembali ke Kriteria & Bobot |
| Perbarui kriteria | `PUT /criteria/{id}` | Kembali ke Kriteria & Bobot |
| Hapus kriteria (soft delete) | `DELETE /criteria/{id}` | Kembali ke Kriteria & Bobot |
| Jalankan perhitungan MFEP | `POST /results/calculate` | Kembali ke Hasil Rekomendasi (ditolak bila bobot ≠ 1.00) |
| Simpan hasil ke arsip | `POST /results/snapshot` | Menuju Detail Arsip yang baru dibuat |
| Hapus arsip | `DELETE /snapshots/{id}` | Kembali ke Arsip Hasil |
